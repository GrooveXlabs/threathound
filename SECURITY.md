# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1.0 | :x:                |

## Reporting a Vulnerability

We take security seriously at GrooveXlabs. If you discover a vulnerability:

1. **Do NOT open a public issue**
2. Email security@groovexlabs.com with:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)
3. We will respond within 48 hours
4. We will keep you updated on our progress
5. We will publicly acknowledge your contribution (with your permission) after the fix is released

## Security Practices

This project follows these security principles:
- Never trust user input
- Least privilege by default
- Defense in depth
- Privacy by design
- Secure by default
- Fail securely

## Security Scanning

This repository is continuously scanned with:
- GrooveGuard (static analysis)
- Dependabot (dependency CVEs)
- GitHub CodeQL (code analysis)
